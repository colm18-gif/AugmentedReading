How it works

The Five-Step Method

From a reader's text selection to an inline visual — the complete patented architecture, explained.

The patented method — Claim 1

Selection to visual in under three seconds.

Every Augmented Reading interaction follows the same five-step process, regardless of the content type, the visual category, or the AI model used to generate the output.

01
Text Selection
The reader highlights any descriptive passage of five words or more within the reading interface. This uses the standard text selection event already present in every web and native reading application — no new interaction model is required. The selected text string is captured and passed to the classification step.
02
AI Classification
The selected text is sent to a classification API — a call to a large language model with a structured prompt instructing it to identify which of the ten visual categories the passage belongs to, and return a confidence score. A cardiac cycle passage returns "anatomical, 0.94." A storm system passage returns "meteorological, 0.91." This call takes under one second in a production implementation.
03
Intelligent Routing
The confidence score determines the output tier. High confidence routes to the full visual generation module. Medium confidence routes to a simplified output. Low confidence or genuinely abstract content routes to a contextual insight — a brief, useful observation rather than a visual. The reader never sees this decision. They see only the result.
04
Visual Generation
The routed request triggers the appropriate generation module. For diagrammatic content — cardiac cycles, molecular pathways, chemical reactions — a parametric renderer produces a precise, accurate animated diagram from the passage parameters. For scene-based content — landscapes, historic sites, architecture — an image generation model produces an AI illustration from the passage text. This is the step the prototype demonstrates across twelve scenes.
05
Inline Display
The visual is rendered inline below the highlighted passage, within the reading interface, with the reader's position preserved. The surrounding text remains in place. When the reader dismisses the visual, the document reflows and they return to exactly the word they were reading. No navigation occurred. No tab was opened. The reader never left the document.
Step four in detail

Three generation approaches.
Always the right output.

The routing step selects not just a visual category but the entire generation pipeline appropriate to that content. Different types of visual require fundamentally different approaches.

Parametric
Animated Diagrams
Pre-built rendering modules populated with passage-specific parameters. Medically and scientifically accurate. Faster and more reliable than generative models for process-based content.
Generative
AI Illustrations
Image generation models produce scene-based visuals from structured prompts derived from the passage. Landscapes, architectural reconstructions, historical scenes.
Insight
Contextual Text
For abstract or low-confidence passages, a language model produces a brief, useful observation. One or two sentences that add comprehension value. Never an error state.
Architecture principles

Model-agnostic. Platform-agnostic.
Content-agnostic.

"The patent protects the method — not the AI model used to implement it. A licensee may use any model for any step. Different models can be used for different visual categories within the same title."
Model-agnostic by design. Claude, GPT-4o, Gemini, Llama, or any future model. The classification step, the generation step, and the parametric renderers can each use a different model optimised for that specific task.
Platform-agnostic. Works across EPUB, PDF and web-based reading interfaces. Implementable at device level, application level, or platform level.
No content modification required. The augmentation layer sits within the reading application. The underlying content file is untouched. The entire existing catalogue of any publisher or platform is immediately compatible.
Graceful degradation. Three output tiers based on classification confidence ensure the system always responds usefully. The reader never encounters an error state.
Reader-initiated only. Nothing appears without the reader's active choice. The feature is completely invisible until wanted — unlike enhanced ebooks, which were always-on and disruptive.
